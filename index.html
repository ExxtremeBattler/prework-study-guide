HTML

  • HTML = HyperText Markup Language
  • HEAD = secret webpage info...
  • BODY = visible content on webpage
  • h1, h2, h3 etc = headers in desc tiers
  • br = line break
  • a = hyperlink
  • div = defines a division/section
  • ol = ordered list
  • table, tr, th = table, rows, column names
  • strong = bold text
  • alt = alt text for img
  • link rel = "stylesheet" type = "text/css" href "filename" = ext. css link
  • style tag = internal css
  • style = "style = styleproperty" = inline css, alter specific props within HTML

CSS

  • margin = space around ele
  • padding = space around inside of ele
  • you can use words or hex codes for colors
  • padding -> border -> margin
  • display: block = block display
  • display: inline = inline
  • display: flex = flexbox (remember to use this on parent ele)
  • align-items : aligns items along main axis
  • justify-content: aligns items along cross axis (the axis other than main)

Git

  • "git status" = current branch
  • "git checkout -b branch-name" = leaves curr branch, creates + joins branch-name
  • "git add -A" / "git add ." = stages all changed files, ready for commitment (lol)
  • git commit -m "what this commit did" = commit w/ message
  • "git pull origin main" = syncs remote main w/ local main, PERFORM OFTEN
  • git push origin 'branch name' = pushes branch up to origin and that branch
  • git push = use only when working solo, just pushes the joint up man
  • git clone "link" = clone repo

JavaScript

  • function = set of instructions, must CALL to use it
  • variable = container for a value (string, number, whatevs)
  • array = big container for multiple values, starts from index 0

Terminal

  • "pwd" = present working direc
  • "cd ~ / cd -" = reset direc to base
  • "cd .. " = move up one lvl
  • "ls" = lists all items in current direc
  • "rm" = remove item
  • "explorer . " = opens current direc in GUI
  • "mkdir foldername " = creates new folder
  • "touch filename" = creates new file
